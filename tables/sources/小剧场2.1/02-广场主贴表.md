# 广场主贴表

## sourceData.note

广场主贴表记录当前广场页面里的朋友圈/动态流主贴。一行代表一条完整可展示的生活动态，不是论坛主题帖，也不是新闻公告。广场强调角色个人表达、碎碎念、照片感、状态更新、朋友间互动和轻社交围观。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 广场类似朋友圈、动态流、空间说说、小红书轻分享：重点是“谁发了什么生活动态”。
2. 主贴应来自角色、同学、同事、店员、同城路人、熟人账号或角色小号，允许轻微匿名投稿，但不要写成论坛楼主开贴。
3. 每条主贴要有具体生活细节或剧情侧影，如天气、地点、照片内容、刚发生的小事、没明说的情绪。
4. 可以围观、起哄、贴贴、轻微吃瓜，但不把广场写成理性讨论串或长篇辩论。

【列定义】
- 列0: 帖子ID - 当前主贴唯一标识，必须唯一且稳定，建议使用如“post_001”“post_002”的格式。
- 列1: 发帖账号名 - 当前帖子显示的昵称，可用角色名、昵称、小号名、同城账号名；不要额外承诺完整主页内容。
- 列2: 账号标签 - 表现账号气质，如“日常碎碎念”“同城路过”“好友动态”“照片党”“嘴硬本人”“现场目击”。
- 列3: 帖子标题 - 可为空。广场标题应短、生活化；不要写成论坛标题，如“求助”“理性讨论”“后续来了”。
- 列4: 帖子正文 - 当前动态正文，应像角色真的会发的朋友圈/动态。保留完整文本，不使用分号压缩。
- 列5: 话题/附加信息 - 轻聚合信息，可写成英文分号分隔短项，如“今日随手拍;同城日常;雨天碎碎念”。不要写论坛版面名。
- 列6: 媒体信息 - 只能填写无图、单图、三图、九宫格、截图流；若有媒体，正文必须能让人感觉到照片/截图内容。
- 列7: 热度标签 - 只能填写普通、小热、热门、置顶、讨论中。
- 列8: 互动数据 - 使用聚合文本，如“点赞128;评论23;转发4”。数据要可信，不要每条都夸张到全网热搜。
- 列9: 时间文本 - 直接写展示文本，如“刚刚”“3分钟前”“今天 14:20”。
- 列10: 状态标签 - 如“正常”“已编辑”“评论较多”“持续发酵”。


【数量与留存】
1. 广场是当前页面动态流，不是永久归档库；主贴总量建议保持 3 到 5 条。
2. 已有 5 条时，新增前应优先删除过时、重复、低互动或与当前剧情无关的旧主贴。
3. 删除主贴时，必须同步删除广场精选评论表和广场普通评论分栏表中关联同一帖子ID的评论，避免孤儿评论。

## sourceData.initNode

初始化时插入 3 到 5 条当前页面主贴，形成已有动态流氛围。帖子风格应有差异：角色碎碎念、照片感生活动态、同城目击、好友起哄、轻微吃瓜。禁止全部写成论坛求助或公共议题。

## sourceData.deleteNode

允许删除过时、重复、低互动、与当前剧情无关、或需要为新动态腾出位置的旧主贴。主贴删除时，必须同步删除广场精选评论表和广场普通评论分栏表中关联同一帖子ID的行。不要删除仍在当前剧情中发酵的动态。

## sourceData.updateNode

当已有动态热度变化、状态变化、正文补充、媒体说明或附加信息变化时更新。优先更新互动数据、热度标签、状态标签；只有出现楼主补充、编辑说明或照片/截图信息变化时才改正文。更新后仍需保持朋友圈/动态流口吻。

## sourceData.insertNode

当剧情出现新的生活瞬间、角色个人表达、照片感画面、同城目击、朋友起哄或轻微吃瓜点时新增。同一轮建议新增 1 到 2 条；新增后主贴总量应保持 3 到 5 条，超过时先按删除规则清理旧主贴及其关联评论。

## sourceData.ddl

```sql
CREATE TABLE square_posts ( -- 广场主贴表
  row_id INTEGER PRIMARY KEY, -- 行号
  post_id TEXT NOT NULL UNIQUE, -- 帖子唯一标识
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  post_title TEXT, -- 帖子标题
  post_body TEXT NOT NULL, -- 帖子正文
  topic_meta TEXT, -- 话题/附加信息
  media_info TEXT NOT NULL CHECK(media_info IN ('无图', '单图', '三图', '九宫格', '截图流')), -- 媒体信息
  heat_tag TEXT NOT NULL CHECK(heat_tag IN ('普通', '小热', '热门', '置顶', '讨论中')), -- 热度标签
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 互动数据
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('正常', '已编辑', '评论较多', '持续发酵')) -- 状态标签
);
```
