# 广场精选评论表

## sourceData.note

记录主贴下最有存在感的短评论。一行一条精选评论；只回应关联主贴，不写论坛长辩论，不替正文补剧情。

【评论方向】
- 评论应围绕主贴内容、照片、楼主状态、公开信息或评论区关系展开。
- 语言必须呈现高强度网感。允许：阴阳怪气、破防反讽、突然发情、开盒威胁、单字定罪、流汗黄豆等 emoji 轰炸。拒绝温和礼貌的捧场，要让评论区充满真实网络骂战的压迫感与荒诞感。
- 可以短、锐、好笑、心疼、起哄、补充、求证、轻微反对，但必须用上述抽象语体包装。
- 所有评论只能基于主贴公开内容和评论者合理视角；禁止全知主角或其他npc行动、私密细节、内心活动。
- 评论要有角色/账号语气差异不要所有人同一种口吻。

【字段规则】
- 关联帖子ID必须对应已存在主贴。
- 评论账号名写真实社交昵称感。
- 账号标签写评论者在这条评论里的气质。
- 评论正文要短而具体，有明确回应点；观点、情绪、调侃、质疑、补充都写进正文，不再拆成标签。正文必须符合上述抽象语体，用空格、emoji、不规则断句。
- 点赞/回复数据和时间文本要可信；被围观、被楼主回应、讨论升温等状态变化写进评论正文或互动文本，不单独归类。

【数量与关联】
每条主贴通常0-2条精选评论，最多3条。优先给最有画面、正在发酵或关系互动强的主贴补评论。主贴删除时同步删除对应评论；不要给已被清理的旧主贴留下孤儿评论。

## sourceData.initNode

初始化时可为部分主贴补少量精选评论，制造已有互动感。评论必须基于主贴公开内容和评论者合理视角；不要围绕主角统一起哄，也不要写成长回复。

## sourceData.deleteNode

主贴删除时必须同步删除对应评论。

## sourceData.updateNode

当精选评论点赞、回复或互动语境变化时更新。正文只在修正错别字、消除歧义或补充公开可见的楼主回应时修改；不要把未公开剧情或全知信息塞进旧评论。

## sourceData.insertNode

当某条主贴出现高存在感时新增。每轮通常1-3条；新增前确认关联帖子ID存在，且同一主贴不要堆太多精选评论。

## sourceData.ddl

```sql
CREATE TABLE square_featured_comments ( -- 广场精选评论表
  row_id INTEGER PRIMARY KEY, -- 行号
  post_ref TEXT NOT NULL, -- 关联帖子ID
  commenter_name TEXT NOT NULL, -- 评论账号名
  commenter_tag TEXT NOT NULL, -- 账号标签
  comment_body TEXT NOT NULL, -- 评论正文
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 点赞/回复数据
  time_text TEXT NOT NULL -- 时间文本
);
```
