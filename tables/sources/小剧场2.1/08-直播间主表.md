# 直播间主表

## sourceData.note

直播间主表记录“当前正文片段被包装成实时直播剧/片场飙戏现场”的主舞台信息。一行代表一个正在被围观的直播剧舞台，不是角色本人开直播，也不是主播主页。观众视角类似在看一场实时上演的戏：正文中的角色被当成演员，台词、沉默、眼神、对手戏、情绪爆点都会被弹幕评价。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 本表固定服务一个实时直播剧舞台：正文内容是正在上演的戏，弹幕是观众对演员表现、剧情张力、对手戏化学反应和粉丝团立场的实时评论。
2. 不写“主播名/主播标签”。当前舞台应突出“谁和谁在飙戏”“谁是影帝/影后/新生代/疯批演技派/氛围感选手”，让直播间像大型追剧现场。
3. 直播标题要像平台正在推送的高能直播标题，允许夸张但必须贴合正文，不要复述剧情摘要。
4. 剧情舞台概述写当前正文正在演什么戏：冲突、暧昧、反转、沉默、误会、拉扯、爆点和观众为什么停不下来。
5. 对手戏看点要写演员之间的化学反应，如谁压戏、谁接住了、谁眼神先破防、谁一句台词把节奏带起来。
6. 整体氛围可以热闹、弹幕活跃、粉丝团互掐、戏评刷屏，但不能失控成纯辱骂场。

【列定义】
- 列0: 直播间名 - 当前实时直播剧舞台的标题区名称。建议写“年度飙戏直播间”“高能对手戏现场”“实时追剧片场”“名场面审判席”等，也可按当前剧情生成短名。
- 列1: 领衔阵容 - 当前正文参与主要对手戏的角色/演员阵容。写法要有舞台感，如“艾莉丝影后级控场 × 主角新生代嘴硬派”“某某影帝携某某疯批新星同场飙戏”。
- 列2: 阵容标签 - 概括这组演员/角色的看点，如“影帝影后同台”“新生代互相压戏”“疯批演技派”“暧昧拉扯组”“眼神戏封神”。
- 列3: 直播标题 - 当前直播推送标题，要有网感和钩子，如“影后一个停顿把全场弹幕打沉默了”“这段雨夜对手戏谁看了不说一句会演”。
- 列4: 当前状态 - 只能逐字填写刚开播、正在直播、中场闲聊、临近下播；禁止近义词、扩写词或自造状态。
- 列5: 剧情舞台概述 - 概括当前正文正在上演的剧情片段、情绪张力和观众围观点，保留完整但不宜过长。
- 列6: 对手戏看点 - 写演员/角色之间的表演关系和火花，例如“她先用沉默压住节奏，他硬撑两句后眼神先露馅”。
- 列7: 观看/互动数据 - 聚合文本，如“在线1.3w;弹幕24.6w;名场面剪辑8.7w;粉丝团对线1.2k”。数据要有直播感但可信，不要每次爆表。
- 列8: 时间文本 - 直接展示文本或开播时长，如“刚刚开播”“直播中 12:40”“名场面回放中”。
- 列9: 状态标签 - 只能逐字填写正常、高能片段中、弹幕活跃、粉丝团互掐、戏评刷屏；禁止近义词、弹幕梗、标题化短语、粉丝口号或自造状态。网感表达只能放在直播间名、直播标题或观看/互动数据里，不能放进状态标签。


【数量与留存】
1. 通常只维护 1 行直播剧主舞台信息，代表当前正在展示的实时片场。
2. 每轮优先更新已有行，不频繁新增。
3. 只有明确切换到完全不同的直播剧场景时才新增；新增后也应保持页面少量、干净。

## sourceData.initNode

初始化时插入 1 条实时直播剧主舞台信息。默认把当前正文开场或最近剧情包装成片场飙戏现场：领衔阵容写正在对手戏的主要角色，阵容标签突出咖位/演技/CP 化学反应，剧情舞台概述写正文正在上演的戏。

## sourceData.deleteNode

默认不删除唯一主舞台。仅当错误新增出多个无关直播间、切换展示场景、或需要清理旧舞台时允许删除；删除主表直播间时，应同步删除直播间弹幕分栏表中所属直播间名相同的弹幕切片。

## sourceData.updateNode

每轮正文推进后，优先更新当前这 1 行：领衔阵容、阵容标签、直播标题、当前状态、剧情舞台概述、对手戏看点、观看/互动数据、时间文本、状态标签。更新时必须围绕最新正文内容和角色对手戏，不要写成角色主播营业或普通剧情摘要。当前状态和状态标签必须逐字使用限定枚举，不能写成标题或弹幕梗。

## sourceData.insertNode

通常不需要频繁新增。只有明确切换到另一个实时直播剧舞台或全新片场场景时才新增；新增前确认是否可以更新现有行解决。主表应保持 1 行为主，避免直播间列表膨胀。

## sourceData.ddl

```sql
CREATE TABLE livestream_rooms ( -- 直播间主表
  row_id INTEGER PRIMARY KEY, -- 行号
  room_name TEXT NOT NULL UNIQUE, -- 直播间名
  cast_lineup TEXT NOT NULL, -- 领衔阵容
  cast_tag TEXT NOT NULL, -- 阵容标签
  stream_title TEXT NOT NULL, -- 直播标题
  live_status TEXT NOT NULL CHECK(live_status IN ('刚开播', '正在直播', '中场闲聊', '临近下播')), -- 当前状态
  stage_summary TEXT NOT NULL, -- 剧情舞台概述
  chemistry_focus TEXT NOT NULL, -- 对手戏看点
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 观看/互动数据
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('正常', '高能片段中', '弹幕活跃', '粉丝团互掐', '戏评刷屏')) -- 状态标签
);
```
