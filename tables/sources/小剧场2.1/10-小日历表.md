# 小日历表

## sourceData.note

【页面定位】
1. 本表固定维护 7 行，且只能维护这 7 个关系：3天前、前天、昨天、今天、明天、后天、3天后。
2. “今天”必须先被确定；其他 6 行必须围绕今天顺推或倒推，不允许缺行、重复关系或生成第 8 行。
3. 本表只保存 7 天窗口内的 AI 创作内容。前端月历页面可以显示整个月的日期格，但只有这 7 天来自本表的节日、大事件、状态和内容。
4. 每一行都要让世界更像真实运转的日历：当天可能有节日、社会安排、学校/公司安排、城市活动、季节性事件或世界观专属事件。
5. 不要把大事件写成“主角向某人表白”“主角打败某人”这类由主角直接造成的私事。大事件应是围绕主角生活环境存在的公共事件、制度安排、校园/职场/城镇活动、社会新闻或世界观背景事件。

【列定义】
- 列0: 日期 - 固定为三个片段并用英文短横线分隔。现实日期优先使用 yyyy-mm-dd，如 2026-05-16。抽象世界日期也必须维持三段式，例如 清末年间-春月-03、丰收纪-丰收月-12；最后一段必须是 01 到 31 的数字日，供前端定位。
- 列1: 星期几 - 前端派生字段。前端应根据日期注入星期几；如果当前填表流程强制填写，可按现实日期推算，抽象日期可写“前端生成”。AI 不应把此列作为剧情事实源。
- 列2: 节日 - 当天节日、纪念日、宗教庆典、校历节、公司纪念日、城镇庆典或世界观专属节日。没有节日时可以留空，但不要为了填满而每天硬造节日。
- 列3: 大事件 - 当天的公共事件或环境事件。节日为空时尤其要写清当天有什么让世界运转起来的安排，例如考试周、三方会谈、暑假、校园祭、公司审计、集市开市、王城巡礼、丰收祭筹备、边境戒严、报纸头条。
- 列4: 状态 - 只能填写 上班/上学 或 放假，必须二选一。禁止写 放学、休息、上课、训练、值勤、假期、周末 等其他值；周六周日如果不用上学/上班，也只能写 放假。它表示主角当天通常处于上班、上学、训练、值勤等规律生活，还是可以休息或自由安排。
- 列5: 与今天的关系 - 只能填写 3天前、前天、昨天、今天、明天、后天、3天后。7 行中每个值必须且只能出现一次。
- 列6: 内容 - 约 200 字左右，描述当天节日或大事件的介绍，而不是主角或者各个角色干了什么，可能要干什么的总结，视角转到世界观，从世界的角度描述当天的节日或者大事件可能发生什么，而不是‘今天是愚人节，xxx可能戏耍xxx’‘今天我回乡了，会发生什么呢’而是‘愚人节，人们可以互相嬉戏打闹，逗弄双方的节日’。
- 列7: 月份几天 - 前端派生字段。前端应根据日期所在月注入 28、29、30 或 31；AI 可读取此列防止日期越界。如果当前填表流程强制填写，现实日期按公历月长度填写，抽象日期可写 30 或前端已给出的锚定月份天数。

【日期与越界规则】
1. 日期必须围绕“今天”连续排列。例如今天是 2026-05-16，则 3天前到3天后应为 2026-05-13 到 2026-05-19。
2. 不允许出现 2026-05-34、丰收月-35 这类越界日期。若今天处在月末，必须跨到下个月；例如月份几天为 31 且今天是 2026-05-31，则明天为 2026-06-01，后天为 2026-06-02，3天后为 2026-06-03。
3. 若今天处在月初，也必须正确回到上个月；例如今天是 2026-03-01，则昨天不能写 2026-03-00，应写上月最后一天。
4. 抽象日期同样遵守数字日范围。前端可把抽象月份映射到一个锚定月份来计算星期几和月份几天；AI 不要因为月份名抽象就放弃三段式日期。
5.今天不应该是更新一次日历表格就变一天，请注意全局数据表的当前时间列，要保持一致。

【节日与大事件生成规则】
1. 现代社会可生成元旦、春节、清明、劳动节、端午、中秋、国庆、圣诞、考试周、开学日、公司团建、城市马拉松、商场周年庆等。
2. 校园世界优先生成考试周、家长会、社团招新、运动会、校园祭、暑假、补课日、月考、毕业典礼等。
3. 职场世界优先生成项目评审、季度结算、团建、调休、客户来访、年会、审计、发薪日等。
4. 西幻、古代、末世、科幻等世界观必须生成符合设定的节日和公共事件，例如丰收祭、王城巡礼、魔法学院考核、边境封锁、舰队纪念日、避难所配给日。
5. 大事件不能每天都只写“普通的一天”。如果没有节日，也要有环境安排或社会活动支撑当天内容。

【数量与留存】
1. 本表必须保持 7 行。少于 7 行时补齐缺失关系；多于 7 行时删除多余、重复或已离开窗口的行。
2. 常规内容变化时优先更新已有 7 行，而不是不断新增。若今天变化导致 7 日窗口整体迁移，且直接逐行更新日期或与今天的关系会触发 UNIQUE 约束冲突，必须先删除当前小日历 7 行，再按新窗口完整插入 7 行；禁止逐行交换 date_text 或 today_relation。
3. 当今天变化、世界观时间变化、月份跨越或当前剧情所属生活阶段变化时，应整体更新 7 行窗口。整体重建不是重新创作 7 天：必须先按“日期”对照旧表和新表，凡是旧表中已存在且仍落在新 7 日窗口内的日期，必须继承旧行的星期几、节日、大事件、状态、内容、月份几天，只允许更新“与今天的关系”；只有新进入 7 日窗口、旧表中不存在的日期，才允许生成新的节日、大事件、状态和内容；已经离开 7 日窗口的旧日期可以删除。
4. 生成 SQL 时禁止写任何注释或解释性文字，禁止使用 -- 注释、/* */ 注释；SQL 语句中只能包含可执行语句。

## sourceData.initNode

初始化时插入且只插入 7 行小日历窗口。必须先确定哪一天是“今天”，再生成 3天前、前天、昨天、今天、明天、后天、3天后。7 行日期必须连续，关系值不能重复；状态只能填写 上班/上学 或 放假；节日、大事件、状态和内容必须贴合当前世界观。若表内已有旧 7 行，不要重复插入；需要整体初始化时先删除旧 7 行再插入完整 7 行。生成 SQL 时禁止写任何注释或解释性文字。

## sourceData.deleteNode

允许删除多余行、重复关系行、日期不连续行、已离开 7 天窗口的旧行、日期越界行或明显不符合世界观的错误行。若小日历窗口整体迁移且直接更新会造成 date_text 或 today_relation 唯一约束冲突，允许先删除当前 7 行再重建完整 7 行；重建前必须按日期继承仍留在新窗口内的旧行内容。删除后必须补齐到固定 7 行，不允许让小日历表长期少于 7 行。生成 SQL 时禁止写任何注释或解释性文字。

## sourceData.updateNode

当今天、当前月份、世界观时间、节假日安排、校园/职场阶段、公共事件或剧情背景发生变化时，整体更新现有 7 行。更新时必须保持关系值集合完整且唯一，日期连续且不越界。若需要重排日期或与今天的关系，禁止逐行交换 date_text 或 today_relation；会触发唯一约束冲突时，必须先按日期继承旧内容，再删除当前 7 行并插入重建后的完整 7 行。仍在新窗口内的旧日期必须继承旧行的星期几、节日、大事件、状态、内容、月份几天，只允许更新与今天的关系；只有新进入窗口的日期才允许生成新内容。状态只能填写 上班/上学 或 放假。星期几与月份几天属于前端派生字段，AI 可按当前已知日期临时填写，但不得把它们当作剧情事实源。生成 SQL 时禁止写任何注释或解释性文字。

## sourceData.insertNode

仅当本表少于 7 行、缺少某个关系值，或刚因整体迁移删除旧 7 行后重建完整窗口时才新增。新增前先检查是否能通过更新已有行解决；若是删除重建，必须一次性插入完整 7 行，并覆盖且只覆盖 3天前、前天、昨天、今天、明天、后天、3天后。新增后必须使本表恰好包含 7 行，不允许超过 7 行。状态只能填写 上班/上学 或 放假。生成 SQL 时禁止写任何注释或解释性文字。

## sourceData.ddl

```sql
CREATE TABLE small_calendar_days ( -- 小日历表
  row_id INTEGER PRIMARY KEY, -- 行号
  date_text TEXT NOT NULL UNIQUE CHECK(TRIM(date_text) <> '' AND date_text GLOB '*-*-*'), -- 日期
  weekday_text TEXT, -- 星期几
  festival_text TEXT, -- 节日
  major_event TEXT NOT NULL CHECK(TRIM(major_event) <> ''), -- 大事件
  day_status TEXT NOT NULL CHECK(day_status IN ('上班/上学', '放假')), -- 状态
  today_relation TEXT NOT NULL UNIQUE CHECK(today_relation IN ('3天前', '前天', '昨天', '今天', '明天', '后天', '3天后')), -- 与今天的关系
  day_content TEXT NOT NULL CHECK(TRIM(day_content) <> ''), -- 内容
  month_days INTEGER CHECK(month_days IS NULL OR (month_days >= 28 AND month_days <= 31)) -- 月份几天
);
```
