# 论坛小组侧栏表

## sourceData.note

论坛小组侧栏表记录论坛页面右侧或侧边的信息块。它不是“更多帖子入口”，而是当前社区版面的氛围、公告、热门话题、活跃状态和版规提醒。适合用单格多条、英文分号分隔的写法压缩非核心正文内容。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 侧栏负责让论坛像真实社区：有版面公告、热门讨论、今日活跃、相关话题和版规提醒。
2. 侧栏内容应短、像列表项，不写成长篇主楼。
3. 栏目内容串必须使用英文分号分隔；每条内容应像侧栏短项目或公告短句。
4. 侧栏服务当前版面窗口，不是永久目录；通常保持 1 到 3 条。

【列定义】
- 列0: 分区/版面名 - 当前侧栏块所属版面，需与论坛主贴表里的版面风格一致。
- 列1: 栏目类型 - 只能填写热门讨论、版面公告、今日活跃、相关话题、版规提醒。
- 列2: 栏目标题 - 当前侧栏块的标题，如“本版热议”“今日围观入口”“发帖前先看”。
- 列3: 栏目内容串 - 一格 3 到 6 条短内容，使用英文分号分隔，由前端拆分。
- 列4: 热度文本 - 如“讨论升温”“今日活跃”“新帖较多”。
- 列5: 时间文本 - 直接展示文本。
- 列6: 状态标签 - 如“正常显示”“公告中”“侧栏更新”。



【数量与关联】
1. 论坛侧栏通常保持 1 到 3 条。
2. 优先更新已有侧栏块，不频繁新增。
3. 如果当前版面切换、旧公告失效或热门方向变化，可删除旧侧栏块。

## sourceData.initNode

初始化时插入 1 到 3 条侧栏块，形成论坛页面结构。建议覆盖热门讨论、版面公告或今日活跃，栏目内容串使用英文分号分隔。

## sourceData.deleteNode

允许删除已过时、与当前版面无关、重复、失效公告或为了控制侧栏数量需要清理的侧栏块。通常保持 1 到 3 条。

## sourceData.updateNode

当栏目内容串、热度文本、时间文本、状态标签变化时更新。优先更新已有侧栏块，保持短列表口吻，不把侧栏写成长帖正文。

## sourceData.insertNode

当版面出现新的热门方向、公告、今日活跃状态、相关话题或版规提醒时新增。无需过多新增，通常保持 1 到 3 条；超过时先删除过时侧栏块。

## sourceData.ddl

```sql
CREATE TABLE forum_sidebar ( -- 论坛小组侧栏表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  section_type TEXT NOT NULL CHECK(section_type IN ('热门讨论', '版面公告', '今日活跃', '相关话题', '版规提醒')), -- 栏目类型
  section_title TEXT NOT NULL, -- 栏目标题
  section_items TEXT NOT NULL CHECK(TRIM(section_items) <> ''), -- 栏目内容串
  heat_text TEXT NOT NULL, -- 热度文本
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('正常显示', '公告中', '侧栏更新')) -- 状态标签
);
```
