# 广场普通评论分栏表

## sourceData.note

广场普通评论分栏表用并列压缩方式表现动态流评论区的滚动密度。一行对应一条主贴下的一组普通评论切片，前端按英文分号拆分为多条评论。它负责营造朋友圈/动态评论区的前排、路人、乐子人与轻微杂音，不承担论坛楼层讨论。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 普通评论应像动态下快速刷出的社交评论：短句、有语气、有关系感、有现场感。
2. 评论必须回应关联主贴的具体内容，不要脱离主贴泛泛夸赞。
3. 允许轻吐槽、贴贴、问号、起哄、补前情、轻微拱火，但杂音只作调味，不主导评论区。
4. 所有评论串必须使用英文分号分隔；不要使用中文分号、顿号或换行冒充分隔。

【列定义】
- 列0: 关联帖子ID - 必须对应当前广场主贴表中已存在的帖子ID。
- 列1: 前排普通评论串 - 一格 3 到 5 条，偏好友回应、贴贴、感叹、接梗、轻吐槽，使用英文分号分隔。
- 列2: 路人评论串 - 一格 2 到 4 条，偏路过围观、补前情、随手评价、求后续，使用英文分号分隔。
- 列3: 杂音/拱火评论串 - 一格 0 到 3 条，偏轻阴阳、乐子人、拱火，恶意攻击，使用英文分号分隔。
- 列4: 时间文本 - 当前这一组评论切片的展示时间。
- 列5: 状态标签 - 如“评论活跃”“路人增多”“节奏轻微上升”。



【数量与关联】
1. 每条主贴通常 0 到 1 行分栏评论切片即可，评论热度特别高时最多 2 行。
2. 优先更新已有评论切片，不要无限新增。
3. 如果关联主贴被删除，必须删除对应分栏评论，禁止留下孤儿评论。

## sourceData.initNode

初始化时可为主贴配 1 到 2 条分栏评论切片，形成评论区已有滚动感。评论应具体回应主贴内容，覆盖前排好友、路人围观和少量乐子人杂音。

## sourceData.deleteNode

允许删除关联主贴已不存在、内容重复、过时、恶意化或为了控制评论密度需要清理的分栏评论。主贴删除时必须同步删除对应分栏。

## sourceData.updateNode

当某一类评论声量变化明显时更新对应评论串与状态标签。优先调整评论串、时间文本与状态标签，而不是频繁新增过多行。更新后仍需使用英文分号分隔。

## sourceData.insertNode

当主贴进入围观状态、评论数明显增多、或需要表现前排滚动感时新增。一条主贴通常只需配 1 行分栏评论切片；新增前确认关联帖子ID存在，并避免同一主贴评论切片过多。

## sourceData.ddl

```sql
CREATE TABLE square_comment_bands ( -- 广场普通评论分栏表
  row_id INTEGER PRIMARY KEY, -- 行号
  post_ref TEXT NOT NULL, -- 关联帖子ID
  front_comments TEXT NOT NULL CHECK(TRIM(front_comments) <> ''), -- 前排普通评论串
  passerby_comments TEXT NOT NULL CHECK(TRIM(passerby_comments) <> ''), -- 路人评论串
  noise_comments TEXT, -- 杂音/拱火评论串
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('评论活跃', '路人增多', '节奏轻微上升')) -- 状态标签
);
```
