# 广场普通评论分栏表

## sourceData.note

用压缩评论串表现主贴下的普通评论密度。一行对应一条主贴的一组评论切片；评论只回应关联主贴，不承担论坛讨论或剧情播报。

【评论方向】
- 前排评论偏熟人、好友、轻互动、接梗、贴贴、吐槽。
- 路人评论偏公开围观、随手评价、求后续、轻微疑问。
- 杂音/拱火只能少量调味，但可包含标准网络恶意：开盒威胁、全家桶警告、性压抑爆发、拱火式复读。不能主导评论区，只作为点缀出现。
- 所有评论只能基于主贴公开内容和评论者合理视角；禁止全知主角或其他npc行动、私密细节、内心活动。
- 评论区要像许多人短促发言，不要写成同一种声音的合唱。

【字段规则】
- 关联帖子ID必须对应已存在主贴。
-  前排普通评论串：3-5条，英文分号分隔；每条必须呈现不同抽象角度避免重复。
- 路人评论串：2-4条，英文分号分隔；体现围观群众的抽象多样性。
- 杂音/拱火评论串：0-3条，英文分号分隔；没有可留空，有则必须是高强度抽象表达，不许温和。
- 禁止用中文分号、顿号或换行代替英文分号。
- 时间文本要可信；评论声量、围观变化、轻微节奏变化都写进评论串或时间语境，不单独归类成标签。

【数量与关联】
每条主贴通常0-1行评论切片，热度高时最多2行。优先更新已有切片，不要无限新增。主贴删除时同步删除对应分栏评论；不要给已被清理的旧主贴留下孤儿评论。

## sourceData.initNode

初始化时可为部分主贴配少量分栏评论切片，形成评论区滚动感。评论应围绕主贴公开内容，声音有差异；不要把评论区写成主角粉丝团或全知围观团。

## sourceData.deleteNode

主贴删除时必须同步删除对应分栏。

## sourceData.updateNode

当评论声量、评论方向或时间语境变化时更新对应评论串。优先更新已有切片，不要频繁新增。更新后仍必须使用英文分号分隔，并保持评论只基于公开可见信息。

## sourceData.insertNode

当主贴评论增多、出现前排滚动、路人围观或少量杂音时新增。一条主贴通常只需1行分栏评论；新增前确认关联帖子ID存在，并避免同一主贴评论切片过多。

## sourceData.ddl

```sql
CREATE TABLE square_comment_bands ( -- 广场普通评论分栏表
  row_id INTEGER PRIMARY KEY, -- 行号
  post_ref TEXT NOT NULL, -- 关联帖子ID
  front_comments TEXT NOT NULL CHECK(TRIM(front_comments) <> ''), -- 前排普通评论串
  passerby_comments TEXT NOT NULL CHECK(TRIM(passerby_comments) <> ''), -- 路人评论串
  noise_comments TEXT, -- 杂音/拱火评论串
  time_text TEXT NOT NULL -- 时间文本
);
```
