# 论坛精选回应表

## sourceData.note

记录论坛主贴下最值得看的楼层回应。一行一条回应；只回应关联主贴，不替正文补剧情，不做朋友圈贴贴，也不当主角直播评论区。


【字段规则】
- 关联帖子标题必须对应已存在主贴。
- 回应账号名写符合网感的网名。
- 账号标签写本条回应气质。
- 回应正文要求如下。
- 有用/回应数据和时间要可信；楼主是否读到、讨论是否继续、回应是否被顶起，都写进正文或互动文本。

【回应方向】
- 你的评论完全融入现代互联网，是真实网络对线的碎片化输出。
- 评论短促、有力、具有高度传染性。
- 语言同样混合抽象嘴臭与集美语体，撕裂感要强。
- 评论可能完全脱离主贴内容，纯粹为了发泄情绪或展示性压抑状态。
- 也可体现贴吧抗压式对线：接梗、反黑、上升家人、翻旧账、假装理中客再阴阳一击。
- 同样禁止正规语言与标点，使用空格、emoji、混乱断句，保持真实的“评论区疯人院”质感。
- 所有评论只能基于主贴公开内容和评论者合理视角；禁止全知主角或其他npc行动、私密细节、内心活动。

【数量与关联】
每条主贴通常3-5条精选回应。主贴删除时同步删除对应回应；不要给已清理主贴留下孤儿楼层。

## sourceData.initNode

初始化时可给部分论坛主贴补多量精选回应，形成楼层讨论感。回应必须基于主楼公开内容和网友合理视角；不要围绕主角统一围观，也不要全都写成严肃长评。

## sourceData.deleteNode

主贴删除时必须同步删除对应回应。

## sourceData.updateNode

当有用数、回应数或楼内互动语境变化时更新。正文只在修正误解、补充主楼公开信息或回应楼主有限更新时修改。

## sourceData.insertNode

当主贴出现值得展示的经验补充或有限蹲后续时新增。每轮通常1-3条；新增前确认关联帖子标题存在，并避免重复同一观点。

## sourceData.ddl

```sql
CREATE TABLE forum_featured_replies ( -- 论坛精选回应表
  row_id INTEGER PRIMARY KEY, -- 行号
  thread_title TEXT NOT NULL, -- 关联帖子标题
  replier_name TEXT NOT NULL, -- 回应账号名
  replier_tag TEXT NOT NULL, -- 账号标签
  reply_body TEXT NOT NULL, -- 回应正文
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 有用/回应数据
  time_text TEXT NOT NULL -- 时间文本
);
```
