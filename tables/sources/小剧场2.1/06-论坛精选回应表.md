# 论坛精选回应表

## sourceData.note

论坛精选回应表记录论坛主贴下最值得阅读、最有楼层感的回应。一行代表一条完整回应，承接认真讨论、礼貌反驳、经验补充、共鸣表达、蹲后续、补充前情或轻微质疑。它不是朋友圈评论，不负责贴贴刷屏。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 精选回应应像论坛楼层：有观点、有信息量、有对主楼或其他楼层的回应。
2. 可以认真补充、礼貌反对、提供经验、贴出观察、提醒楼主、蹲后续、轻微抬杠。
3. 允许一点论坛冷幽默和阴阳怪气，但不能变成持续骂战。
4. 回应者可为版面常驻、匿名用户、老用户、补资料的人、理中客、吃瓜群众。

【列定义】
- 列0: 关联帖子标题 - 必须对应当前论坛主贴表中已存在的帖子标题。
- 列1: 回应账号名 - 当前回应显示昵称，应像论坛账号或半匿名用户。
- 列2: 账号标签 - 用于表现回应者气质，如“版面常驻”“认真回复”“补充资料”“理中客”“蹲后续”“经验用户”。
- 列3: 回应正文 - 完整保留，不做压缩。应有明确观点或补充信息，不要只有“同意”“蹲”“哈哈”。
- 列4: 回应立场 - 只能填写共鸣、补充、礼貌反对、经验分享、轻微质疑。
- 列5: 有用/回应数据 - 聚合文本，如“有用34;回应9”。
- 列6: 时间文本 - 直接展示文本。
- 列7: 状态标签 - 如“高赞回应”“楼主已读”“讨论继续中”。



【数量与关联】
1. 每条论坛主贴通常配 1 到 3 条精选回应；不要给同一帖子堆满重复观点。
2. 同一轮新增精选回应建议 1 到 3 条；优先补给正在持续讨论或楼主更新的帖子。
3. 如果关联主贴被删除，必须删除对应精选回应，禁止留下孤儿楼层。

## sourceData.initNode

初始化时可给论坛主贴补 2 到 4 条精选回应。回应应覆盖共鸣、补充、礼貌反对、经验分享或轻微质疑，形成真实楼层讨论感。

## sourceData.deleteNode

允许删除关联主贴已不存在、重复、过时、恶意化、或没有信息量的精选回应。主贴删除时必须同步删除对应回应。

## sourceData.updateNode

当有用数、回应数、状态变化时更新。若楼主已读、观点被顶上来、讨论继续发酵，可更新状态标签与互动数据；正文只在补充上下文、修正误解或回应楼主追加信息时修改。

## sourceData.insertNode

出现值得展示的补充、共鸣、礼貌反驳、经验分享、轻微质疑、蹲后续或楼层金句时新增。每轮建议 1 到 3 条；新增前确认关联帖子标题存在，并避免重复同一个观点。

## sourceData.ddl

```sql
CREATE TABLE forum_featured_replies ( -- 论坛精选回应表
  row_id INTEGER PRIMARY KEY, -- 行号
  thread_title TEXT NOT NULL, -- 关联帖子标题
  replier_name TEXT NOT NULL, -- 回应账号名
  replier_tag TEXT NOT NULL, -- 账号标签
  reply_body TEXT NOT NULL, -- 回应正文
  stance_tag TEXT NOT NULL CHECK(stance_tag IN ('共鸣', '补充', '礼貌反对', '经验分享', '轻微质疑')), -- 回应立场
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 有用/回应数据
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('高赞回应', '楼主已读', '讨论继续中')) -- 状态标签
);
```
