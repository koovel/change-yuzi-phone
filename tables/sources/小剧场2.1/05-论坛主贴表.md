# 论坛主贴表

## sourceData.note

论坛主贴表记录当前论坛/贴吧/社区板块里的主题帖主楼。一行代表一条完整主楼，重点是公共讨论氛围、版面感、楼主叙述、半匿名社区语气和可继续回复的讨论点。它不是朋友圈动态流，不强调发帖人的个人主页，也不写成照片碎碎念。row_id 由框架自动维护，无需生成；下面的列编号从业务列开始计数。

【页面定位】
1. 论坛类似贴吧、社区小组、校园墙、树洞、同城论坛、经验交流区：重点是“一个话题被拿出来讨论”。
2. 帖子应有主楼感：楼主说明背景、提出问题、吐槽经历、整理经验、八卦爆料或记录后续。
3. 账号可以匿名、半匿名、老用户、小号、版面常驻、楼主本人；不要全部写成现实角色本人发朋友圈。
4. 允许争议、求助、吃瓜、经验贴、避雷、蹲后续、轻微抬杠；不写成恶意骂战。

【列定义】
- 列0: 分区/版面名 - 当前帖子所属的版面，如“生活闲谈”“同城互助”“情感树洞”“校园墙”“避雷交流”“都市怪谈”。
- 列1: 发帖账号名 - 当前主楼作者昵称，可用“匿名楼主”“盐汽水小号”“南门路过人”“版面老咸鱼”等论坛感昵称。
- 列2: 账号标签 - 用于表现作者气质，如“匿名楼主”“老用户”“认真整理”“经验分享”“求助中”“蹲后续本人”。
- 列3: 帖子类型 - 只能填写求助、吐槽、长文分析、观后感、生活记录、经验贴。
- 列4: 帖子标题 - 必须成立，承担论坛入口吸引力。标题要像社区帖子，如“有没有人也遇到过这种突然递伞的情况”“记录一下今晚便利店门口那件事”。
- 列5: 帖子正文 - 完整保留，不做压缩。正文应交代具体背景和讨论点，比广场动态更清楚、更有主楼叙述感。
- 列6: 附加信息 - 轻聚合信息，可写成英文分号分隔短项，如“楼主补充中;含后续更新;评论区求图”。
- 列7: 热度/回应数据 - 聚合文本，如“回应57;有用128;收藏39”。
- 列8: 时间文本 - 直接展示文本。
- 列9: 状态标签 - 如“正常”“持续讨论”“楼主更新”“友好交流中”。



【数量与留存】
1. 论坛展示当前版面窗口，不是永久归档库；主贴总量建议保持 3 到 5 条。
2. 已有 5 条时，新增前应删除已沉帖、重复、低价值、过时或没有继续讨论价值的旧帖。
3. 删除主贴时，必须同步删除论坛精选回应表中关联同一帖子标题的回应，避免孤儿楼层。

## sourceData.initNode

初始化时插入 3 到 5 条论坛主贴，形成已有版面氛围。帖子类型应有差异，如求助、吐槽、经验贴、生活记录、长文分析；标题必须像论坛入口，不要写成朋友圈动态。

## sourceData.deleteNode

允许删除已沉帖、重复、过时、低价值、恶意化、或需要为新讨论腾出位置的旧主贴。删除主贴时必须同步删除论坛精选回应表中关联同一帖子标题的回应。不要删除仍在持续讨论或有楼主更新价值的帖子。

## sourceData.updateNode

当楼主补充、热度变化、状态变化、附加信息变化时更新。可更新“楼主补充中”“含后续更新”“争议点转移”等附加信息；正文只在楼主补充、修正描述或追加后续时修改。更新后仍需保持论坛主楼语气。

## sourceData.insertNode

出现新讨论点、求助需求、经验总结、八卦爆料、避雷内容、版面热帖或公共争议时新增。每轮建议新增 1 到 2 条；新增后论坛主贴总量应保持 3 到 5 条，超过时先按删除规则清理旧帖及其关联回应。

## sourceData.ddl

```sql
CREATE TABLE forum_threads ( -- 论坛主贴表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  thread_type TEXT NOT NULL CHECK(thread_type IN ('求助', '吐槽', '长文分析', '观后感', '生活记录', '经验贴')), -- 帖子类型
  thread_title TEXT NOT NULL UNIQUE, -- 帖子标题
  thread_body TEXT NOT NULL, -- 帖子正文
  meta_text TEXT, -- 附加信息
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 热度/回应数据
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('正常', '持续讨论', '楼主更新', '友好交流中')) -- 状态标签
);
```
