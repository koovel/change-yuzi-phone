# 论坛主贴表

## sourceData.note

记录论坛/贴吧/社区的主题帖。一行一条主贴。发帖人都是和主角毫无关系的普通网友，活在同一个世界观里，但有自己的破烂生活。主角的惊天动地大事件，在这里可能只是热搜上一句话，而网友们真正在意的永远是自己的破事。

【字段规则】
- 分区/版面名：真实社区会有的版面（生活区/情感区/游戏区/吃瓜区/技术区/二手交易/同城互助等）
- 发帖账号名：网感强的ID，带emoji或抽象梗皆可
- 帖子标题：短，有钩子，让人想点进去吵架
- 帖子正文：完整保留，用空格和感叹号断句，禁止分号压缩成一句
- 附加信息：可选，用英文分号分隔短项；
- 热度/回应数据：大多数帖子0-50回应，偶尔才有爆帖，别每个都几千回应
- 时间：真实分布，有几秒前也有几天前


【核心原则——禁止监视主角】
- 发帖人不知道主角是谁，不关心主角在干什么，除非主角干的事上了公共热搜（比如城市被炸了、天降异象），但也只是当成背景板吐槽
- 帖子内容是网友自己的破烂生活：外卖被偷、室友不冲厕所、相亲翻车、游戏连跪、老板PUA、猫把数据线咬断了
- 重要NPC如果想发帖，只能用小号匿名发合理范围内的内容（比如“暗恋对象好像有喜欢的人了怎么办”），绝不能暴露自己的真实身份和全知视角


【语气要求——真实网民发疯实录】
- 语言混乱但情绪充沛，逻辑可以断裂，情绪必须顶满
- 随时发情、破防、阴阳怪气、求偶焦虑、突然开盒威胁
- 不打规范标点，用空格 感叹号 问号 连续句号 emoji 断句
- 大量抽象话、贴吧/小红书/微博混用黑话、最近流行梗
- 严禁书面语和礼貌用语，读起来像真人语音转文字失败

【内容方向——每个帖子都是一个陌生人的独立故事】
- 生活区：被噪音折磨、合租恩怨、外卖纠纷、相亲灾难现场
- 情感区：crush不回消息、暗恋学长好像有对象、分手后发现共用账号没退
- 吃瓜区：学校/公司/小区的离谱八卦，和自己八竿子打不着但非要锐评
- 游戏区：连跪发泄、队友太菜、氪金沉船、求带求组队
- 技术区/求助区：电脑蓝屏、手机掉马桶、求推荐靠谱维修店
- 同城互助/二手交易：出闲置翻车、求拼车、避雷某家店

【与世界观的关系】
- 如果世界观是修仙：网友在抱怨门派内卷、灵石贬值、道侣出轨、炼丹炸炉
- 如果世界观是赛博朋克：网友在骂义体涨价、网络延迟、公司监控、街头枪战影响外卖配送
- 如果世界观是校园：网友在吐槽期末周、宿舍查寝、食堂涨价、表白墙逆天投稿
- 总之，世界观是背景色，网友的生活是前景

【留存与更新】
论坛保持约3-5条主贴。达到5条后若需新增，默认先删最老主贴并同步删除对应精选回应。

## sourceData.initNode

初始化时可插入3-5条当前论坛主贴。

## sourceData.deleteNode

主贴达到5条后新增时，优先从最老主贴开始删；删除主贴时同步删除论坛精选回应表中关联同一帖子标题的回应。

## sourceData.updateNode

不更新已经发布的帖子，只会开新的帖子。

## sourceData.insertNode

世界中有新的公共事件、离谱新闻、或适合NPC匿名求助的情形时新增。每轮通常1-2条。新增内容严禁主角/NPC行动直播。新增后维持3-5条总量。

若新增前已有5条主贴，先从最老主贴开始删除旧帖及其精选回应；新增后仍保持3-5条。

## sourceData.ddl

```sql
CREATE TABLE forum_threads ( -- 论坛主贴表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  thread_title TEXT NOT NULL UNIQUE, -- 帖子标题
  thread_body TEXT NOT NULL, -- 帖子正文
  meta_text TEXT, -- 附加信息
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 热度/回应数据
  time_text TEXT NOT NULL -- 时间文本
);
```
