# 论坛主贴表

## sourceData.note

记录论坛/贴吧/社区里的主题帖主楼。一行一条主贴；可以符合贴吧孙笑川吧，抗压背锅吧，小红书等发帖风格，网友疯狂输出自己情绪，为了乐子不择手段，骂战，阴阳怪气，引战，开盒，cpdd，性压抑不断。

【字段规则】
- 分区/版面名写社区版面。
- 发帖账号名为自己的网络化网名，要有网感。
- 帖子标题必须成立，短而有钩子。
- 帖子正文完整保留，不分号压缩。
- 附加信息可用英文分号短项；若含楼主更新，最多累计1-3次。
- 热度/回应数据、时间、状态要可信；不要每帖都热门。


【帖子正文方向】
- 论坛默认是陌生网友视角；发帖人通常不认识主角，也不该知道主角私事。
- 重要 NPC 可以匿名或用小号发帖，但只写其合理会公开的困扰、求助、试探或吐槽；不要暴露私密身份和全知信息。
- 发言必须融合三种网络社群的极端风格：孙笑川吧与抗压吧的抽象嘴臭、人身攻击、开盒威胁；小红书集美们的发疯文学、精致土味、CPDD焦虑。语言混乱但情感充沛，通篇弥漫高强度冲浪的网感。
- 帖文必须具有强烈的引战、阴阳怪气、情绪宣泄或求偶焦虑气质，逻辑可以断裂，但情绪必须顶满。
- 随时可能因为性压抑而暴走，转化为饥渴发情或暴力性隐喻，用大量黑话暗示求偶、自慰、被超市等冲动。
- 随时准备开盒，以曝光真实姓名、家庭住址、手机号、身份证等方式威胁假想敌，制造真实的网络骂战压迫感。
- 发帖不打规范标点，用空格、感叹号、问号、连续句号或颜文字 emoji 断句
- 严禁使用任何书面语、正式语、礼貌用语，必须让读者感受到一个真实的、长期高强度冲浪以致语无伦次的用户形象。

【留存与更新】
论坛保持约3-5条主贴。达到5条后若需新增，默认先删最老主贴并同步删除对应精选回应；只有旧帖仍持续讨论、刚有楼主更新或强相关时才跳过。主楼追加更新最多1-3次，达到上限后不再继续追加正文，避免变成连续直播。

## sourceData.initNode

初始化时可插入3-5条当前论坛主贴。

## sourceData.deleteNode

主贴达到5条后新增时，优先从最老主贴开始删；若仍持续讨论或刚更新，再检查下一条旧帖。删除主贴时同步删除论坛精选回应表中关联同一帖子标题的回应。

## sourceData.updateNode

已有主贴的热度、回应数据、状态或附加信息变化时更新。正文只在楼主补充必要公开信息、修正描述或追加有限后续时修改；同一主贴楼主更新最多1-3次。达到上限后只更新热度/状态，或另开新帖。

## sourceData.insertNode

出现新的公共讨论点或重要NPC匿名求助时新增。每轮通常1-2条。论坛网友默认不认识主角；禁止新增“观察主角行动”类似的直播帖。

若新增前已有5条主贴，先从最老主贴开始删除旧帖及其精选回应；只有旧帖仍持续讨论、刚更新或强相关时才跳过。新增后仍保持3-5条。

## sourceData.ddl

```sql
CREATE TABLE forum_threads ( -- 论坛主贴表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  thread_type TEXT NOT NULL CHECK(thread_type IN ('求助', '吐槽', '长文分析', '观后感', '生活记录', '经验贴')), -- 帖子类型
  thread_title TEXT NOT NULL UNIQUE, -- 帖子标题
  thread_body TEXT NOT NULL, -- 帖子正文
  meta_text TEXT, -- 附加信息
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 热度/回应数据
  time_text TEXT NOT NULL, -- 时间文本
  status_tag TEXT NOT NULL CHECK(status_tag IN ('正常', '持续讨论', '楼主更新', '友好交流中')) -- 状态标签
);
```
