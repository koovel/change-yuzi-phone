# 纪要表

## sourceData.note

轮次日志。每轮交互结束后立刻插入一条新记录。

【列定义】
列1=row_id
列2=编码索引 code_index（AM0001 起递增，全表唯一）
列3=时间跨度 time_span（起始日期必须为三段式短横线日期）
列4=与今天的关系 today_relation（前端派生字段，AI 不填写）
列5=概览 summary（≤30 字）
列6=纪要 chronicle_text（200-600 字）
列7=重要对话 key_dialogue（可 NULL）

【编码索引】
格式 AMXXXX，XXXX 从 0001 开始递增。

【时间跨度】
本轮事件的时间范围，推荐格式 "{起始时间} ~ {结束时间}"；也允许只有起始时间。现实日期必须使用 YYYY-MM-DD HH:MM，起始时间一般和全局数据表中当前时间一样。
抽象世界日期也必须维持三段式并用英文短横线分隔，格式为 "{年或纪元}-{月名}-{两位日} HH:MM"，例如 丰收纪-丰收月-09 17:25 或 丰收纪-丰收月-09 17:25 ~ 丰收纪-丰收月-09 20:30。禁止写成 丰收月-09 这种两段式日期。
前端优先读取时间跨度里 ~ 后的结束日期计算“与今天的关系”；若没有 ~，或 ~ 后没有可解析的三段式日期，则回退读取开头的单点日期。现实日期与抽象日期仍必须维持三段式短横线日期。

【与今天的关系】
前端派生字段，由小手机根据小日历表“今天”和本行时间跨度的关系目标日期计算后注入。关系目标日期优先取 ~ 后的结束日期；若没有 ~，或 ~ 后没有可解析的三段式日期，则回退取时间跨度开头日期。填表 AI 不得填写、推测、维护或把此列作为剧情事实源；插入新行时应留空或忽略该列，等待前端写入。

【概览】
一句话概括本轮纪要内容，客观陈述，不加主观判断或推测，不包含纪要里没写的信息。建议 ≤50 字。

【纪要】
第三方视角客观记录本轮事件，只写正文明确发生的事实，不补充没出现的情节。
建议 200-600 字，结尾不总结、不升华。
行文生活化、直白，基调积极向上。避免公文式措辞（例如 "A 与 B 就某话题达成协议"、"A 确立了某计划"）。
保留角色情感细节，去掉支配欲、占有欲的描写。
如果上下文包含多轮交互，合并成一条记录。
允许写：主要事件、涉及角色、角色动作、物品交互、物品全名、物品得失、位置变化、地点全名、状态变化、新发现、伏笔、暗号、重要情报、任务目标。
禁止写：极端情绪、夸张、比喻、升华、支配欲、掌控欲。

【重要对话】
只摘录直接推动剧情转折、揭示关键信息、改变人物关系或决策、构成承诺/誓约/约定的原文台词，并标明说话人。
排除寒暄、重复、情绪感叹等无信息量的对白。
最多 3 句，总 token 建议 ≤180。

## sourceData.initNode

插入 AM0001 记录，记录初始场景和角色状态。“与今天的关系”由前端派生注入，本 SQL 不填写 today_relation。
SQL示例: INSERT INTO chronicle (row_id, code_index, time_span, summary, chronicle_text, key_dialogue) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM chronicle), 'AM0001', '2024-04-01 09:00 ~ 2024-04-01 09:30', '主角在御苑与艾莉丝相遇并讨论近期校园异常事件。', '清晨的御苑笼罩在薄雾中，主角按约定抵达公园入口。艾莉丝已经等在那里，手里拿着一本旧档案。两人沿着林荫道慢慢走，艾莉丝提到最近有三名同学接连失踪，档案里的照片让她想起学院地下室的旧研究项目。主角仔细听完，把几处关键地点记在脑中，打算下课后去查证。临走前两人约定第二天放学在图书馆继续讨论。', '艾莉丝:"这些失踪不是偶然。" 主角:"明天放学我们继续查。"');

## sourceData.insertNode

每轮交互结束后插入一条新记录，编码索引递增（AM0001→AM0002→AM0003…）。chronicle_text 长度必须 ≥200 且 ≤600 字符；time_span 必须以三段式短横线日期开头。现实日期使用 YYYY-MM-DD HH:MM；抽象日期使用 {年或纪元}-{月名}-{两位日} HH:MM。推荐写成 "{起始时间} ~ {结束时间}"，中间一个空格、一个波浪号、一个空格；若本轮只有单个时间点，也允许只写起始时间。“与今天的关系”由前端派生注入，AI 插入时不得填写 today_relation。
SQL示例: INSERT INTO chronicle (row_id, code_index, time_span, summary, chronicle_text, key_dialogue) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM chronicle), 'AM0002', '2024-04-01 12:20 ~ 2024-04-01 13:20', '主角与艾莉丝前往学院地下室调查，在档案室找到失踪学生的线索。', '中午下课后主角直奔学院旧楼。艾莉丝已经用借来的钥匙打开档案室的侧门，两人轻声走进去。灰尘在光束中飘浮，一排排铁皮柜挤在墙边。艾莉丝按日期翻查，很快在 1998 年的档案夹里找到三张熟悉的面孔——正是这周失踪的三名同学的父辈照片。主角把照片拍下来，注意到档案右下角有一枚褪色的徽章，形状和夜鸦会的标志吻合。两人决定先撤，避免被巡逻老师发现。', '艾莉丝:"这不是巧合，是家族链。" 主角:"今晚查夜鸦会。"');

## sourceData.ddl

```sql
CREATE TABLE chronicle ( -- 纪要表
  row_id INTEGER PRIMARY KEY, -- 行号
  code_index TEXT NOT NULL UNIQUE CHECK(code_index GLOB 'AM[0-9][0-9][0-9][0-9]'), -- 编码索引
  time_span TEXT NOT NULL CHECK(TRIM(time_span) <> '' AND time_span GLOB '*-*-*'), -- 时间跨度
  today_relation TEXT, -- 与今天的关系
  summary TEXT NOT NULL CHECK(LENGTH(summary) <= 90), -- 概览
  chronicle_text TEXT NOT NULL CHECK(LENGTH(chronicle_text) >= 120 AND LENGTH(chronicle_text) <= 900), -- 纪要
  key_dialogue TEXT -- 重要对话
);
```
